## Spec

- 本项目是针对 [soco-cli](https://github.com/avantrec/soco-cli) 工具包的 Docker 镜像构建，旨在提供一个便捷的环境来使用 soco-cli 进行 Sonos 设备的管理和控制。
- 镜像基于官方的 Python 最新稳定版本镜像构建 (Python >= 3.13)，确保兼容性和性能。
- 镜像中预装了 soco-cli 工具包，每次构建时使用最新版本，用户可以直接使用该工具包进行 Sonos 设备的发现、控制和管理，无需额外安装。
- 镜像设计为轻量级，尽可能减少不必要的依赖和文件，以提高构建速度和运行效率。
- 镜像提供了一个默认的入口点，使用户可以直接运行 soco-cli 命令来与 Sonos 设备进行交互。
- 镜像支持环境变量配置，目前支持日志级别 (`LOG_LEVEL`) 的配置。
- 镜像支持 soco-cli 的 Interactive Shell Mode，使用户可以在容器内直接进入交互式命令行界面，方便进行设备管理和调试。
- 镜像支持在容器内运行 soco-cli 的 SoCo-CLI HTTP API Server 模式，默认端口为 8000，支持通过命令行参数配置端口，使用户可以通过 HTTP 请求来控制 Sonos 设备，适合与其他系统集成。
- 镜像支持挂载外部配置文件和数据卷，包括配置目录和本地音乐库路径，以便用户可以持久化配置和数据。
- 镜像会初始化示例配置文件，帮助用户快速了解配置结构。
- 容器以非 root 用户运行，确保安全性。
- 镜像仅支持 arm64 架构。
- 镜像标签采用语义化版本命名规则（如 `latest`、`v1.0.0` 等）。
- 镜像定期更新，以确保包含最新的 soco-cli 版本和相关依赖，同时修复已知的安全漏洞和性能问题。
- 镜像提供一个标准 Docker Compose 文件示例，方便用户在 Docker Compose 环境中部署和管理该镜像。
- README.md 文件中包含了详细的使用说明、示例命令和常见问题解答，帮助用户更好地理解和使用该镜像。
- DESIGN.md 文件中包含了镜像的设计原则、架构图和技术细节，帮助开发者理解镜像的内部结构和实现细节。
